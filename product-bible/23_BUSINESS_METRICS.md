# Purpose

This document defines the business metrics used to measure the success of Kharcha.

The objective is to ensure product decisions are guided by meaningful user outcomes rather than vanity metrics.

These metrics help answer questions such as:

- Are users receiving value?
- Are users building healthy financial habits?
- Is the product growing sustainably?
- Which features create the most engagement?
- What should the team improve next?

---

# Measurement Principles

Every metric should be:

- Actionable
- Measurable
- Reliable
- Consistent
- User-focused

Avoid tracking metrics that do not influence product decisions.

---

# Product Vision

Kharcha exists to improve financial awareness.

Success is measured by users developing better financial habits—not simply by using the application more frequently.

---

# North Star Metric

## Financial Awareness Score (FAS)

Definition

The percentage of active users who consistently:

- Record expenses
- Stay within budget
- Review insights
- Improve Financial Health

Formula

```

Financial Awareness Score =
Users meeting healthy financial behavior criteria
─────────────────────────────────────────────── × 100
Monthly Active Users

```

Why it matters

A higher Financial Awareness Score indicates that users are actively managing their finances rather than simply opening the application.

# Acquisition Metrics

Measure how users discover Kharcha.

| Metric | Description |
|--------|-------------|
| App Installs | Total installations |

---

# Activation Metrics

Activation measures whether users experience the core value of Kharcha.

A user is considered **Activated** when they:

- Complete onboarding
- Set a monthly budget
- Add their first expense
- View the dashboard

Primary Metric

Activation Rate

Formula

```

Activated Users
─────────────── × 100
New Users

```

Target

≥ 70%

---

# Engagement Metrics

Measure how actively users interact with Kharcha.

| Metric | Target |
|--------|--------|
| Daily Active Users (DAU) | Increasing |
| Weekly Active Users (WAU) | Increasing |
| Monthly Active Users (MAU) | Increasing |
| DAU / MAU Ratio | ≥ 30% |
| Average Session Duration | 2–5 Minutes |
| Sessions Per Week | ≥ 3 |

# Habit Metrics

The goal is to create healthy financial habits.

| Metric | Target |
|--------|--------|
| Weekly Expense Logging | ≥ 70% of active users |
| Budget Review Frequency | Weekly |
| AI Insight Views | Increasing |
| Financial Health Checks | Monthly |

Habit formation is more valuable than high session duration.

# Retention Metrics

Retention indicates long-term product value.

| Metric | Target |
|--------|--------|

| Day 7 Retention | ≥ 40% |

# Decision Framework

Before building any feature, ask:

- Does it improve Financial Awareness?
- Will users benefit regularly?
- Can success be measured?
- Does it align with the North Star Metric?

If the answer is **No**, reconsider the feature.

# CEO Vision

Downloads do not build companies.

Habits do.

Kharcha succeeds when people become more aware of their finances, make better spending decisions, and continue returning because the product consistently helps them improve.

Every metric should answer one question:

**"Are we genuinely improving our users' financial lives?"**

If a metric cannot guide a better product decision, it should not be tracked.
